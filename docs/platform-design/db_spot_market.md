# Database Structure: Spot Market App

All tables used by `apps/spot-market/app.py` and its ingestion pipeline
(`apps/spot-watcher/pipeline.py`).

---

## Schema Overview

```
public
└── spot_daily                    ← core price table (main app data source)

staging
├── spot_interprov_flow           ← inter-provincial trading rows (per PDF)
├── spot_report_summaries         ← AI-generated daily market summaries
├── spot_report_documents         ← PDF source registry (knowledge pool)
├── spot_report_pages             ← per-page raw text
├── spot_report_chunks            ← chunked text for retrieval
├── spot_report_facts             ← structured extracted facts
└── spot_report_notes             ← generated Obsidian note registry
```

---

## public.spot_daily

The primary price table. One row per (date, province). All price values are
in **¥/kWh**.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `report_date` | date NOT NULL | Trading day |
| `province_cn` | text | Chinese name e.g. `山东` |
| `province_en` | text NOT NULL | English name e.g. `Shandong` |
| `da_avg` | numeric | Day-ahead average clearing price |
| `da_max` | numeric | Day-ahead max price |
| `da_min` | numeric | Day-ahead min price |
| `rt_avg` | numeric | Real-time average clearing price |
| `rt_max` | numeric | Real-time max price |
| `rt_min` | numeric | Real-time min price |

**Unique constraint:** `(report_date, province_en)`

**Upsert behaviour:** `COALESCE` — an incoming `NULL` never overwrites an
existing non-null value. A partial row (DA only, RT only) is safe to upsert
without clobbering the other side.

**Written by:** `services/spot_ingest/db_upsert.py` → `upsert_rows()`  
**Read by:** `apps/spot-market/app.py` → `load_all()`, `load_kpis()`, `load_provinces()`

## staging.spot_interprov_flow

Inter-provincial spot trading summary, parsed from the
**省间现货交易情况** table in each daily PDF.

One row per `(report_date, direction, metric_type)`.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `report_date` | date NOT NULL | |
| `direction` | text NOT NULL | `送端` (exporting) or `受端` (importing) |
| `metric_type` | text NOT NULL | `最高均价` / `最低均价` / `最高电量` / `最高价` |
| `province_cn` | text | Province with the highest/lowest value for this metric |
| `province_share` | numeric(5,2) | Market share annotation e.g. `35.00` for "(35%)" |
| `price_yuan_kwh` | numeric(8,4) | Clearing price in ¥/kWh |
| `price_chg_pct` | numeric(8,4) | Day-on-day % change |
| `time_period` | text | Active trading window e.g. `09:00–11:15 13:15–14:30` |
| `total_vol_100gwh` | numeric(16,4) | Direction subtotal in 亿kWh; populated for `最高均价` row only |
| `source_pdf` | text | Originating PDF filename |
| `created_at` | timestamptz | |

**Unique constraint:** `(report_date, direction, metric_type)`  
**Upsert behaviour:** COALESCE on price and volume columns; direction/province always replaced.

**Written by:** `services/spot_ingest/interprov_upsert.py` → `upsert_interprov_rows()`  
**Parsed by:** `services/spot_ingest/interprov_parser.py` → `parse_interprov()`

## staging.spot_report_summaries

AI-generated (Claude) daily market narrative. One row per date; re-running
for the same date replaces the row.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `report_date` | date NOT NULL UNIQUE | |
| `summary_text` | text NOT NULL | 2–3 paragraph English narrative |
| `model` | text | Model ID e.g. `claude-sonnet-4-6` |
| `prompt_tokens` | int | |
| `completion_tokens` | int | |
| `source_pdf` | text | Originating PDF filename |
| `created_at` | timestamptz | |

**Written by:** `services/spot_ingest/interprov_upsert.py` → `upsert_summary()`  
**Generated by:** `services/spot_ingest/ai_summary.py` → `generate_summary()`

## staging.spot_report_documents

Source registry for ingested PDFs. Hash-based dedup ensures each physical file
is only processed once regardless of how many times the pipeline runs.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `source_path` | text NOT NULL | Absolute or repo-relative file path |
| `file_name` | text NOT NULL | |
| `report_year` | smallint NOT NULL | |
| `report_date_min` | date | Earliest date found in this PDF |
| `report_date_max` | date | Latest date (multi-day reports) |
| `file_hash` | text NOT NULL UNIQUE | SHA-256 hex |
| `file_size_bytes` | bigint | |
| `page_count` | int | |
| `ingest_status` | text | `pending` / `parsed` / `failed` / `skipped` |
| `parser_version` | text | Default `v1` |
| `parse_error` | text | First 500 chars of error if status = `failed` |
| `created_at` | timestamptz | |
| `updated_at` | timestamptz | |

**Written by:** `services/knowledge_pool/document_registry.py`

## staging.spot_report_pages

Full extracted text for every page of every ingested PDF. Preserved for
re-parsing without re-reading the file.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `document_id` | bigint FK → `spot_report_documents.id` (CASCADE) | |
| `page_no` | smallint NOT NULL | 1-indexed |
| `page_date` | date | Inferred date for this page (nullable) |
| `extracted_text` | text | Full page text from pdfplumber |
| `char_count` | int | |
| `extraction_method` | text | Default `pdfplumber` |
| `created_at` | timestamptz | |

**Unique constraint:** `(document_id, page_no)`

## staging.spot_report_chunks

Overlapping 500-character text chunks for retrieval. Stable `chunk_index`
within a document enables idempotent re-generation.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `document_id` | bigint FK → `spot_report_documents.id` (CASCADE) | |
| `page_no` | smallint | Source page |
| `chunk_index` | int NOT NULL | Sequential within document |
| `chunk_text` | text NOT NULL | |
| `chunk_type` | text | `body` / `table` / `header` / `reason` |
| `report_date` | date | Carried from page date |
| `created_at` | timestamptz | |

**Unique constraint:** `(document_id, chunk_index)`  
**FTS index:** GIN on `to_tsvector('simple', chunk_text)` for Chinese-compatible full-text search.

## staging.spot_report_facts

Typed key-value facts extracted from reports — prices, volumes, drivers.
Acts as a structured layer on top of raw chunks.

| Column | Type | Notes |
|---|---|---|
| `id` | bigserial PK | |
| `document_id` | bigint FK → `spot_report_documents.id` (CASCADE) | |
| `report_date` | date NOT NULL | |
| `province_cn` | text | NULL = national-level fact |
| `province_en` | text | |
| `fact_type` | text NOT NULL | `price_da` / `price_rt` / `driver` / `volume` / `interprovincial` / `summary` |
| `metric_name` | text | e.g. `da_avg`, `rt_max`, `reason_text` |
| `metric_value` | numeric | NULL for text facts |
| `metric_unit` | text | e.g. `yuan/MWh`, `GWh` |
| `fact_text` | text | Source sentence or phrase |
| `page_no` | smallint | |
| `confidence` | text | `high` / `medium` / `low` |
| `source_method` | text | `pdf_regex` / `spot_daily_bridge` / `filename_inference` |
| `created_at` | timestamptz | |

**Unique constraint:** `(document_id, report_date, province_cn, fact_type, metric_name)`

## staging.spot_report_notes

Registry of generated Obsidian-compatible markdown notes. One row per logical
note; points to the file path on disk.

| Column | Type | Notes |
|---|---|---|

| `note_type` | text NOT NULL | `daily_report` / `province` / `concept` |
| `note_key` | text NOT NULL | e.g. `2025-07-28`, `蒙西`, `新能源出力下降` |
| `note_path` | text NOT NULL | Repo-relative path to `.md` file |
| `note_title` | text | |
| `report_date_min` | date | |
| `report_date_max` | date | |
| `generated_at` | timestamptz | |
| `updated_at` | timestamptz | |

**Unique constraint:** `(note_type, note_key)`

## Data Flow

```
PDF file on disk
      │
      ▼
pipeline.run()
      │
      ├─ Step 1: pdf_parser.py         → parsed prices (in memory)
      │
      ├─ Steps 2–3: excel_sync.py      → cross-check vs Excel
      │
      ├─ Step 4: db_upsert.py          → public.spot_daily
      │
      ├─ Step 5: excel_sync.py         → Excel file (fill blanks only)
      │
      ├─ Step 6: interprov_parser.py   → staging.spot_interprov_flow
      │           ai_summary.py        → staging.spot_report_summaries
      │
      └─ Step 7: knowledge_pool/       → staging.spot_report_documents
                                          staging.spot_report_pages
                                          staging.spot_report_chunks
                                          staging.spot_report_facts
                                          staging.spot_report_notes
                                          (+ Obsidian .md files on disk)
```
